### 2. WF1-A 연속 디버깅 — 3단계 오류 해결

#### 오류 ①: `체크리스트 입력 없음` (Execution #127 이전)
- **원인:** 체크 항목 없이 전송 → 입력 검증 코드 정상 동작
- **해결:** 실제 체크 항목 입력 후 재전송

#### 오류 ②: `Invalid Value` — `location: "q"` (Execution #127)
- **원인:** `searchMethod: "name"` 모드에서 전체 Google Drive 쿼리문을 `queryString`에 넣음
  - `name` 모드의 `queryString` = 파일명 텍스트만 입력해야 함
  - `query` 모드의 `queryString` = 전체 Drive API 쿼리문 입력
- **수정:** 3개 Drive search 노드 모두 `searchMethod: "query"` 로 변경

#### 오류 ③: `File not found: .` — `location: "fileId"` (Execution #128, #134)
- **원인:** n8n Google Drive v3 노드의 `searchMethod: "query"` 버그
  - `files.list?q=...` 를 호출해야 하는데 `files.get(fileId=".")` 를 잘못 호출
  - `filter: {}` (비어있음) → driveId 미설정 → URL 빌드 시 `.` 로 처리됨
- **수정:** `searchMethod: "name"` + `filter.folderId` (mode: "id") 조합으로 변경
  - 환자 폴더 확인: `queryString={{ $json.raw.patient_code }}`, `folderId: 1awOFEQ8U2gjCdUWv6BXN3OGUe9CeeVv`
  - interviews 폴더 확인: `queryString="interviews"`, `folderId: {{ $json.folderId }}`
  - 기존 파일 조회: `queryString="interview_"`, `folderId: {{ $json.interviewsFolderId }}`

#### 오류 ④: Credential 오염 (Execution #133)
- **원인:** UI에서 credential 수동 변경 시도 → 존재하지 않는 ID `6CRBPbtmCW7AtsvJ` 로 오염
- **수정:** MCP로 6개 Google Drive 노드 credential 일괄 복원 → `fcKtWyui68XESQND`

```

  searchMethod: "query"

  → n8n 버그: files.get(fileId=".") 호출 → 404

✅ 올바른 방식:
  searchMethod: "name"
  queryString: "PT-2026-001"  (파일명만)
  filter.folderId: { __rl: true, value: "FOLDER_ID", mode: "id" }
```
