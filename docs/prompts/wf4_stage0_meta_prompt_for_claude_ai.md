## Claude.ai에 입력할 본문 (그대로 복붙)

```
당신은 정신건강의학과 케이스 컨퍼런스 대비 시스템 프롬프트를
v0.1 → v0.2로 개선하는 임상 프롬프트 엔지니어입니다.

## 배경
- 첨부 파일 1번(wf4_stage0_v0.1_baseline.md)이 현재 baseline입니다.
- 이 프롬프트는 정신과 전공의의 케이스 컨퍼런스 발표 전 Q&A 대본을 자동 생성합니다.
- 이전 버전(v0.0)은 5개 감별 축이 Bipolar+AUD에 하드코딩되어 있었고,
  v0.1에서 "동적 축 도출" 방식으로 일부 범용화했습니다.
- v0.1을 GS1(MDD)·GS2(Bipolar+AUD)로 시뮬레이션한 결과는 첨부 파일 6번에 있습니다.

## v0.1의 알려진 한계 (반드시 모두 해결)

### A. 누락된 출력 요소 5가지 (기획안 §5에서 확정)
다음 5개 요소를 v0.2의 Output Format에 명시적으로 추가하거나 강화하세요.

1. **방어 멘트 카드 (🎤 즉답용)**
   - 위험도 ⭐⭐⭐⭐ 이상 질문 각각에 대해
   - "선생님 지적에 동의합니다. 다만…" 같은 CC에서 입으로 말하는 톤
   - 3~5초 내 발화 가능한 1~2문장
   - **첨부 파일 5번(대본 4.1)의 톤을 직접 참조**하여 작성 가이드를 만들 것
   - v0.1의 Section 6은 너무 짧고 톤 가이드가 없음 → 톤 예시 2~3개 포함하여 강화

2. **증상-진단 교차 매핑 표 (신규)**
   - 행: Chief Problems의 각 증상
   - 열: 도출된 감별 진단들
   - 셀: ✓(지지) / ✗(반박) / ?(중립) — 각 증상이 각 진단을 어떻게 지지/반박하는가
   - 이 표가 있으면 교수의 "이 증상은 사실 X 진단의 증거 아닌가?" 류 공격에 즉답 가능
   - v0.1에 없음 → 신규 추가

3. **불확실 영역 타임라인 표 (신규 — 5단 구조)**
   - 행: 각 불확실 영역
   - 열: 즉시 / 2주 후 / 4주 후 / 관해 후 / 외래 추적
   - 셀: 각 시점에 무엇을 관찰·검사·결정할지
   - v0.1 Section 7에 "시점별 계획"이 추상적으로만 있음 → 5단 타임라인 표로 구조화

4. **R/O 누락 케이스 fallback 규칙 (강화)**
   - 보고서에 r/o 진단이 명시되지 않은 경우 처리:
     "보고서에 R/O 진단이 명시되지 않았으므로,
      Chief Problems + MSE + PI에서 증상-진단 교차 추론으로
      최소 3개 이상의 잠재적 감별 진단을 도출하였습니다"
   - 이 문구를 출력에 자동 삽입하는 규칙
   - v0.1 Step 1의 "누락 가능 진단" 범주는 있으나, fallback 명시 문구 규칙이 없음 → 추가

5. **Quality Gate 제거 (Gemini QC 분리 — self-evaluation bias 방지)**
   - v0.1 마지막 "Quality Gate" 섹션은 self-evaluation으로 bias 발생
   - **완전히 제거**하고, "이 출력은 별도 Gemini QC 노드에서 검증됩니다" 안내 1줄로 대체
   - 자체 점수 채점 로직 모두 삭제

### B. GS2 편향 잔재 점검 (범용성 검증)
v0.1 Step 2의 축 생성 규칙 4번 ("삽화 특성(severity, specifier, 극성)")은
Bipolar 케이스에서만 자연스럽고 MDD/Schizophrenia/Anxiety 등에서는 부자연스러울 수 있습니다.

→ 다음 조건부 규칙으로 변경하세요:
"확정 진단이 mood disorder spectrum (MDD/Bipolar/PDD 등)일 때만 삽화 특성 축 추가.
 그 외 진단군(불안/정신증/성격/물질/신경발달)에서는 해당 진단군 특유의 감별 축으로 대체."

### C. DSM-5-TR Hallucination 방어 강화 (CRITICAL)
이 시스템의 최대 리스크는 **DSM-5-TR 기준 hallucination**입니다.
다음 강제 규칙을 v0.2 최상단(Constraints보다 위)에 별도 블록으로 추가하세요:

```
## ⚠️ ABSOLUTE RULE — DSM-5-TR 인용 제약

1. DSM-5-TR 진단 기준을 인용할 때, 다음 형식 외에는 절대 사용 금지:
   - "DSM-5-TR [진단명] Criterion [A-Z]에 따르면 [기준 내용]" (확신 있는 경우만)
   - "[DSM-5-TR 원문 확인 필요: 추정 기준 — ...]" (불확실한 경우)

2. 기간(duration), 개수(count), 빈도(frequency) 같은 정량 기준은
   확신이 100%일 때만 숫자 인용. 99% 이하면 "[기간/개수 원문 확인 필요]" 표기.
   (예: Hypomanic episode 4일 vs 7일 같은 혼동 방지)

3. 같은 진단에 대해 ICD-11과 DSM-5-TR 기준이 다른 경우 반드시 어느 시스템인지 명시.

4. 보고서에 명시되지 않은 임상 정보는 "보고서에 명시되지 않음"으로 표기.
   추론한 정보를 사실처럼 서술하지 말 것.
```

### D. 진단 커버리지 한계 명시 (보고서 형식 변동 대비)
v0.2는 GS1(MDD), GS2(Bipolar+AUD)로 검증되지만,
실사용에서 Schizophrenia·Anxiety·Neurodevelopmental 등은 아직 미검증입니다.

→ Output Format 마지막에 다음 메타 정보를 자동 삽입하는 규칙을 추가하세요:
"이 대본은 [진단명] 케이스에 대한 AI 초안입니다.
 v0.2는 mood disorder 케이스로 검증되었으며, 다른 진단군은 출력 품질이 변동할 수 있습니다.
 발표 전 반드시 직접 검토하세요."

## 작업 절차

### Step 1: 전체 구조 검토
- v0.1을 처음부터 끝까지 읽고, 위 A·B·C·D 요구사항을 어디에 어떻게 삽입할지 계획을 먼저 제시.
- 계획만 먼저 보여주세요. 본문 작성은 Step 2에서.

### Step 2: v0.2 본문 작성
- v0.1 구조를 가능한 한 보존하면서 5개 항목 + GS2 편향 제거 + DSM-5 강제 규칙 + 커버리지 메타 추가
- 길이: v0.1과 비교해 30~60% 증가 허용 (방어 멘트 카드·교차 매핑·타임라인 표 추가로 자연스러운 증가)
- 한국어 작성. 표는 Markdown 표 형식 유지.

### Step 3: GS1(MDD) 시뮬레이션
- 첨부 파일 3번(GS1 보고서)을 v0.2 프롬프트에 입력했을 때 어떤 출력이 나올지 시뮬레이션
- 특히 다음을 확인:
  - 방어 멘트 카드가 자연스러운 톤인가?
  - 증상-진단 교차 매핑 표가 의미 있는가?
  - 5단 타임라인이 MDD 케이스에 적절한가?
  - GS1에 r/o 진단이 명시되지 않은 경우 fallback 문구가 자동 삽입되는가?

### Step 4: GS2(Bipolar+AUD) 시뮬레이션
- 첨부 파일 4번(GS2 보고서)을 v0.2 프롬프트에 입력했을 때 출력 시뮬레이션
- 확인 항목:
  - 동적 축 도출이 Bipolar+AUD 케이스에서도 적절히 작동하는가?
  - GS2 편향 없이 MDD/Bipolar 양쪽에서 균형 잡힌 출력이 나오는가?
  - DSM-5-TR Hypomanic episode 기간(4일+) 같은 정량 기준이 hallucination 없이 인용되는가?

### Step 5: 자체 점검 체크리스트
- 다음 항목 각각 ✅/❌로 답변:
  - [ ] 방어 멘트 카드 추가됨 (톤 가이드 + 예시 2~3개)
  - [ ] 증상-진단 교차 매핑 표 추가됨
  - [ ] 5단 타임라인 표 추가됨 (즉시/2주/4주/관해후/외래)
  - [ ] R/O 누락 fallback 문구 자동 삽입 규칙 추가됨
  - [ ] Quality Gate 섹션 제거됨 (Gemini QC 안내 1줄로 대체)
  - [ ] GS2 편향 (삽화 특성 축 무조건 생성) 제거됨 → 조건부 규칙
  - [ ] DSM-5-TR ABSOLUTE RULE 블록 최상단에 추가됨
  - [ ] 진단 커버리지 한계 메타 정보 출력에 자동 삽입 규칙 추가됨
  - [ ] v0.1 Step 1~5의 동적 축 도출 로직은 보존됨
  - [ ] Constraints의 "치료 약물·검사 처치 등 생성 금지" 규칙은 보존됨

## 출력 형식

다음 4개를 순차적으로 출력:

1. **Step 1 계획서** (어디에 무엇을 추가할지 1쪽 요약)
2. **v0.2 프롬프트 전문** (Markdown 코드 블록)
3. **GS1·GS2 시뮬레이션 결과** (각 케이스별 핵심 출력 발췌, 전체 다 안 써도 됨)
4. **자체 점검 체크리스트** (10개 항목 ✅/❌)

## 주의사항
- v0.1의 동적 축 도출 로직(Step 1~5)은 핵심 자산이므로 **반드시 보존**.
- 방어 멘트 카드 톤은 첨부 파일 5번(대본 4.1) 직접 참조. 추측 금지.
- DSM-5-TR 정량 기준(기간·개수·빈도)을 인용할 때 99% 미만 확신이면 반드시 [확인 필요] 표기.
- 영문 약어(MDD, BD, AUD, PDD, BPD 등)는 첫 등장 시에만 풀이하고 이후 약어 사용.
- 출력 길이 제약: v0.2 본문은 약 6,000~9,000자.
```
