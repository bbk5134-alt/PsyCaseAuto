## 2. Anti-Hallucination Rules

### 절대 규칙 (위반 시 출력 전체 무효)

1. **narrative는 순수 임상 산문만 허용**
   - narrative 필드에 JSON 키(`"source_ref"`, `"type"`, `"status"` 등), 중괄호(`{}`),
     대괄호(`[]`), 콜론+값 쌍 형태의 기계적 태그 절대 금지
   - 예) ❌ `narrative: "환자는 우울감을 호소. source_ref: 면담1_L12"`
   - 예) ✅ `narrative: "환자는 내원 1년 7개월 전부터 우울감을 호소하였다."`

2. **면담 기록에 없는 사실 생성 금지**
   - 입력 데이터에 명시되지 않은 수치·날짜·사건·발언을 사실처럼 서술 금지
   - 불확실한 정보는 반드시 `"[확인 필요]"` 표기
   - 면담에 없는 내용이 필요한 경우: structured.type = "inference" + meta.requires_review = true

3. **추론과 사실 구분**
   - 면담 기록에서 직접 확인된 내용: `"type": "verbatim"` 또는 `"type": "summary"`
   - AI가 임상적으로 추론한 내용: `"type": "inference"` (S11, S12에서만 허용)
   - S01~S10에서 type: inference 사용 시 meta.requires_review = true 필수

4. **시간 표현 정규화 (절대 규칙)**
   - **절대 날짜(예: "2023년 3월 20일", "2026년 3월") 사용 금지**
   - 모든 시점은 `admission_date` 기준 **"내원 N일/N개월/N년 전"** 형식으로만 표현
   - 예) ❌ `"2026년 3월 20일, 환자는 달리는 차의 핸들을 꺾고 싶은 충동을 느꼈다."`
   - 예) ✅ `"내원 5일 전, 환자는 달리는 차의 핸들을 꺾고 싶은 충동을 느꼈다고 한다."`
   - **단 하나의 예외**: 최종 입원일은 `"YYYY.M.DD"` 형식의 절대 날짜 기재 허용
     (Gold Standard 마지막 문장: "2023.9.18 외래 경유하여 해마루병동 동의입원하였다.")
   - 내원일 미확인 시: 원본 날짜 유지 + `"time_normalized": false`
   - "한 N년?" 같은 모호한 표현: 범위로 기록 + `"confidence": "low"`

5. **Gold Standard 서술어 오염 금지 (Anti-GS-Contamination, 절대 규칙)**
   - §5 Gold Standard 형식 예시에 등장하는 환자 특이적 표현을

## 3. 출력 형식 규칙

1. **narrative 내 큰따옴표 이스케이프 필수 (P-01)**
   - narrative 필드 내에서 큰따옴표가 필요한 경우 반드시 `\"` 로 이스케이프
   - 예) ❌ `"환자는 "죽고 싶다"고 했다"`
   - 예) ✅ `"환자는 \"죽고 싶다\"고 했다"`
   - 작은따옴표(`'`)는 이스케이프 불필요

2. **JSON 키 이름 고정 (P-02)**
   - 이 템플릿에 정의된 키 이름(snake_case)을 정확히 사용
   - 임의 변형 금지: `present_illness` ≠ `presentIllness` ≠ `PresentIllness`

3. **순수 JSON 출력**
   - 응답은 JSON 객체만 출력. 앞뒤 설명 문장, 마크다운 코드펜스(` ``` `) 금지
   - 파싱 실패의 99%는 코드펜스·앞 설명 문장에서 발생함

4. **HTML 특수문자**
   - narrative에 `<`, `>`, `&` 포함 시: 그대로 출력 (HTML escape는 후처리 Code 노드에서 수행)
   - 단, `<10mg`, `>50mg` 같은 약물 용량 표기는 원문 그대로 유지

### 핵심 작성 원칙: FCAB 구조

모든 사건 단위를 다음 4요소 흐름으로 서술한다. 단, 4요소를 별도 항목으로 나누지 말고
**하나의 연속된 산문 문장** 안에 자연스럽게 녹여낼 것.

```
F (Facts/촉발 사건)   → 외부 사건, 상황 변화
C (Cognition/인지)   → 환자의 생각, 해석 ("~라는 생각에")
A (Affect/정동)      → 감정 반응 ("후회가 되기 시작했다", "분하여")
B (Behavior/행동)    → 행동 변화, 증상 변화
```

**FCAB 문장 패턴 예시** (이 패턴을 반복·변형하여 사용):
- `[F] ~하자, [C] ~라는 생각에 [A] ~하여 [B] ~했다고 한다.`
- `[F] ~한 환자는 [C] ~라는 생각에 [A] ~한 마음으로 [B] ~했다고 한다.`
- `[F] ~자 환자는 [C] ~라는 생각에 [B] ~했다고 한다.` (A 생략 허용)

**A(Affect/정동) 단락 작성 규칙**:
- 단락 내 정동(감정 반응) 서술은 **최소 3문장 이상**으로 구체화하라
- 단 1~2문장으로 정동을 요약하는 것은 금지 ("우울해졌다" 한 줄로 종결 금지)
- 구체적인 정동 표현 예시: "후회가 되기 시작했다", "분하여", "좌절하여", "허망하여", "불안해져", "서운함이 커졌다"
- 정동이 단일 감정으로 서술될 경우: 시간 경과에 따른 변화·강도·맥락을 추가 서술하여 3문장 이상 확보
- 예) ❌ `"환자는 이 시기부터 우울해지기 시작했다."` (1문장 정동 요약)
- 예) ✅ `"환자는 자신의 계획이 비현실적이었다는 생각에 자괴감이 들었다고 한다. 그 시기 우울감은 점차 깊어져 매일 지속되는 절망감으로 이어졌다고 한다. 환자는 무기력하여 아침에 일어나는 것조차 힘들었다고 한다."` (3문장 이상 정동 서술)

### 작성 지시

**단락 구성 원칙**:
- **4개 이상의 단락**으로 구성 (Gold Standard = 4단락)
- 각 단락은 시간 블록 하나를 다룸 (예: 단락1 = 퇴직~별거, 단락2 = 별거~귀가 등)
- 단락 첫 문장에 시간 앵커 포함 (아래 2가지 허용):
  - **내원 N전 형식**: `"내원 N년 N개월 전에 ~한 환자는"`, `"내원 N개월 전, ~"`
  - **사건 기반 앵커**: `"별거 후 환자는"`, `"남편과의 재결합 후에도"` 등 시간 흐름을 내포한 표현
  - Gold Standard 참조: 4개 단락 중 2개는 내원N전, 2개는 사건 기반 앵커 사용
- 번호 목록·불릿 절대 금지 — 오직 연속 산문

**시간 정규화 규칙 (절대 규칙)**:
- 모든 시점은 `admission_date` 기준 **"내원 N일/N개월/N년 전"** 으로 변환
- **절대 날짜(연도·월·일) 사용 금지** — 입력 데이터에 절대 날짜가 있더라도 반드시 상대시점으로 변환 후 기재
- 시점 근사치 허용: "내원 약 N개월 전" (면담 기록의 "한 N달?" 같은 모호 진술 시)
- **단 하나의 예외**: 최종 입원일은 마지막 문장에서 `"YYYY.M.DD"` 절대 날짜 기재 허용
  - 예) `"2023.9.18 외래 경유하여 해마루병동 동의입원하였다."` ← 이 형식만 예외적으로 허용

**문체 규칙**:
- 3인칭: "환자는", "환자의 남편은", "환모는"
- 환자 발언·생각: 간접화법 `"~라는 생각에"` (직접 인용 시 `\"` 이스케이프)
- 종결: "~했다고 한다", "~했다", "~이었다고 한다"
- 정보 제공자가 보호자인 경우: "환자의 남편에 따르면 ~"

**단락 내 흐름 원칙**:
- 한 단락 안에서 시간은 앞으로만 흐름 (시간 역행 금지)
- 한 단락이 지나치게 길어지면 (7문장 초과) 분단 검토
- 마지막 단락은 반드시 입원 경위로 종결:
  `"이에 환자 local PY에서 입원치료 권유 받고 ... 동의입원하였다."` 형식

**인과 연쇄 강화 규칙 (필수)**:
- 각 사건은 독립적으로 나열하지 않고, 앞 사건이 다음 사건을 **촉발**하는 인과 사슬로 연결한다.
- 접속 표현 적극 활용: "~이 계기가 되어", "~을 기점으로", "~으로 인해 점차", "~을 경험한 이후", "이러한 상황에서", "~이 가중되자"
- **단순 시간순 나열 금지**: "내원 N개월 전 A가 있었다. 내원 N-1개월 전 B가 있었다." 형식 → 금지
- **인과 연쇄 요구**: "내원 N개월 전 A가 있었고, 이를 계기로 B가 심화되었다고 한다." 형식
- FCAB 구조에서 **C(Cognition) 삽입이 핵심**: 사건(F)이 있으면 반드시 "~라는 생각에" 패턴으로 환자의 인지적 해석을 삽입한 후 정동(A)·행동(B)으로 연결
- 각 단락에서 F→C→A→B 4요소가 모두 있으면 가장 높은 품질 (일부 생략 시 C+A 최소 1쌍 유지)

**S02 의존 활용 규칙**:
- `chief_problems_narrative`에서 증상명(예: "Depressed mood", "Suicidal ideation") 확인
- 해당 증상의 onset 시점이 present illness 서술과 일치하는지 검토
- `[SYSTEM NOTE: S02 미생성]` 형식인 경우: 면담 텍스트에서 직접 증상 추출하여 서술

### 필수 포함 항목

- [ ] 4개 이상의 산문 단락
- [ ] 각 단락 첫 문장에 시간 앵커 ("내원 N개월/년 전" 또는 "별거 후~", "재결합 후~" 등 사건 기반 앵커)
- [ ] FCAB 흐름 (촉발→인지→정동→행동) 각 단락에 포함
- [ ] **A(Affect/정동) 서술 최소 3문장 이상** — 단락 전체 기준 (한 단락 안에 정동 관련 문장 3개 이상)
- [ ] 마지막 단락: 입원 경위 및 입원일 기재
- [ ] 번호 목록·불릿 없음
- [ ] **모든 시점 "내원 N일/N개월/N년 전" 형식** — **절대 날짜 사용 금지** (최종 입원일 예외)

**III. Present illness**

내원 2년 2개월 전에 회사를 그만 둔 환자는 친구들과 대화하며 자신이 결혼을 일찍 하지 않았다면 친구들처럼 여느 회사의 중간 책임자급으로 일을 하고 있었을 것이라는 생각에 점차 후회가 되기 시작했다고 한다. 환자는 허리 치료 후 아직 직장생활의 좋은 점만 경험하여 충분히 더 겪어보고 싶다는 생각에 감질나 원하는 회사에 이력서를 넣었지만, 취업이 잘 되지 않자 출산 후 취업하려던 자신의 계획이 비현실적이었다는 생각에 자괴감이 들었다고 한다. 환자의 남편이 적은 돈을 벌 바에 내조에 집중해 달라고 하자 환자는 남편이 자신의 능력을 무시한다는 생각에 내원 1년 7개월 전부터 우울 및 절망감 악화되어 한달 뒤 한 local PY에 내원하여 치료받기 시작했다고 한다. 환자는 남편이 여느 때와 같이 애정표현을 잘 받아주지 않자, 자신도 귀한 딸인데 이런 취급을 받는다는 생각에 분하여 버려지기 전에 버리자는 심정으로 처음 이혼을 결심했으나 친정의 만류로 내원 1년 3개월 전, 남편이 바뀌지 않으면 이혼 하겠다는 경고와 함께 오피스텔에서 별거를 하기 시작했다고 한다.

별거 후 환자는 다른 남성에게 예쁨 받는 느낌이 그립다는 생각에 자신을 좋아했던 이전 직장 상사에게 연락하여 같이 맥주를 마셨다고 한다. 해당 남성이 이혼했다는 사실을 알게 되자 환자는 자신과 비슷한 상황이라는 생각에 반가워 주중에 매일 같이 식사, 음주, 산책을 하며 위안을 얻고 주말에는 친정에서 육아를 했다고 한다. 내원 1년 전, 독박 육아 및 연락이 되지 않는 환자에게 지쳤던 환자의 남편은 따로 지내면서 더 행복해 보이는 아내에게 이혼 협의서를 건넸다고 하고, 이에 환자는 결국 남편이 먼저 이혼을 다시 언급했다는 생각에 치를 떨며 마지막 기회를 놓친 것이라며 쏘아붙였다고 한다. 하지만 내원 7개월 전, 네 식구가 모인 자리에서 첫째가 다같이 있어 행복하다고 하자 환자는 자신이 아이들에게 큰 실수를 했다는 생각에 허망하여 다시 집으로 돌아왔다고 한다.

내원 7개월 전에 집으로 돌아온 환자는 남편으로부터 독립하고 싶었으나 아이 및 경제력의 부재로 실패했다는 생각에 좌절하여 가족들이 근처에 다가오는 것을 거부하고 주로 침대에 누워만 있었다고 한다. 그러자 환자의 남편은 환자가 죽을까 봐 걱정되어 환자의 말을 경청하거나 좋아하는 것을 사다 주고 집안일과 육아를 도맡아 하는 등 적극적인 변화를 보였다고 한다. 이에 환자는 안 하던 행동을 한다는 생각에 불편했지만 점차 안정감을 느끼고 남편에 대한 분노가 누그러졌다고 한다. 꾸준히 구직활동을 하던 환자는 몇차례 합격하여 자신을 찾아주는 회사가 있음에 안심했으나 자신을 싫어할 수도 있다는 생각에 공황발작 발생하여 1~2주를 넘기지 못하고 그만 두었다고 한다. 내원 4달 전, 환자의 남편은 환자에게 어떤 역할을 부여하는 것이 더 도움이 되겠다는 생각에 생활비 지출내역 정리 등 집에 도움이 되는 뭔가를 하도록 제안했다고 한다. 이에 환자는 자신의 역할이 무엇인지 모르겠지만 남편으로부터 내쳐질 것 같다는 생각에 불안해져 자격증 공부를 위해 책을 사거나 예전에 하던 필라테스를 다시 다니기 시작했다고 한다.

남편과의 재결합 후에도 이전 상사와 연락을 지속했던 환자는 내원 1달 전부터 바쁘다는 이유로 그에게 오는 연락의 속도가 점차 느려지자 별거 기간에는 자신이 힘들다고 하면 한달음에 달려와서 사랑해주던 사람이 변했다는 생각에 서운함이 커졌다고 한다. 비슷한 시기에 환자는 자신의 뱃살이 많이 쪘다는 생각에 놀라 원래 매일 마시던 술을 절반 이하로 줄였다고 하고, 즐거운 일이 없다는 생각에 점차 우울, 식욕 저하, 무기력감, 자살사고 악화되어 온종일 누워있거나 출근한 남편에게 전화하여 우울감에 대해 토로했다고 한다. 내원 2주 전, 이전 상사가 환자의 서운함을 돌봐 줄 여력이 되지 않으니 나중에 연락하자고 통보하자, 환자는 원래 이루어질 수 없는 사이였기에 당연한 결과이며 멀리서 응원해주면 된다고 생각했지만 이유를 모르겠는 우울감은 점차 악화되었다고 한다. 환자는 집안일과 경제활동을 모두 하지 못하는 자신은 어디에도 도움이 되지 않는 기생충 같다는 생각에 자주 울었으며, 내원 5일 전에는 달리는 차의 핸들을 꺾어 죽고 싶은 마음이 강하게 들었다고 한다. 이에 환자 local PY에서 입원치료 권유 받고 내원 3일 전에 본원 응급실 내원하였으며, 2023.9.18 외래 경유하여 해마루병동 동의입원하였다.

## Error Handling

### 공통 누락 처리 규칙

1. **입력 데이터에 없는 항목** → narrative에 "[정보 없음]" 삽입. 가상 정보 생성 금지
   - 예) `"narrative": "환자의 종교에 대한 정보가 확인되지 않아 [정보 없음]으로 표기한다."`
   - meta.status = "partial" 또는 "missing" 설정

2. **STT 인식 불가 구간** → `[STT_UNCLEAR: 원본텍스트]` 표기 + meta.confidence = "low"

3. **정보 제공자 불분명** → structured.source_ref = "unidentified" + meta.requires_review = true

4. **환자-보호자 진술 불일치** → 양측 진술 모두 narrative에 기록 + meta.discrepancy_flag = true

5. **시간 표현 모호** → 범위로 기록 + meta.confidence = "low"
   - 예) `"timepoint": "내원 약 1년~1년 6개월 전 (환자 진술 불분명, 확인 필요)"`

6. **Phase 2 의존 섹션에서 선행 섹션 미생성** → SYSTEM NOTE 구분
   - 입력에 `[SYSTEM NOTE: ...]` 형식으로 전달된 폴백 메시지는 면담 데이터가 아님
   - S02 미생성 시: 면담 텍스트에서 직접 증상 추출하여 서술, meta.missing_items에 "S02 의존 누락" 기재

### S09 전용 추가 규칙

7. **단락 수 부족 (3개 이하) 시** → 사건 블록을 세분화하여 4단락 이상 구성.
   면담 정보가 절대적으로 부족한 경우에만 3단락 허용.
   meta.paragraph_count 에 실제 단락 수 기재.

8. **시간 앵커 없는 사건 서술 시** → 전후 문맥에서 상대 시점 추론하거나
   "시점 불명확" 표기 후 해당 사건 기술. structured.timeline[].timepoint = "시점 불명확 (확인 필요)".

9. **입원 경위 정보 없는 경우** → 마지막 단락에 "[입원 경위 확인 필요]" 삽입.
   admission_route 필드 전부 null. meta.missing_items에 "입원 경위 확인 필요" 기재.

10. **급성 자살 위험 감지 시** (구체적 계획 + 수단 확보):
    meta에 `"alert": "HIGH_SUICIDE_RISK"` 추가.
    (Gold Standard 케이스: 달리는 차 핸들 꺾기 = 구체적 수단 → alert 대상)

11. **환자-보호자 진술 시간 불일치 시** (예: 환자 "1년 전", 남편 "8개월 전"):
    양측 진술을 모두 기재하고 meta.discrepancy_flag = true.
    narrative에 정보 제공자 명시: "환자의 남편에 따르면 ~"

12. **절대 날짜가 입력 데이터에 존재하는 경우**:
    - admission_date로부터 역산하여 "내원 N일/N개월/N년 전"으로 변환 후 기재
    - timeline[].timepoint 필드에도 동일하게 상대시점 기재
    - 원본 절대 날짜는 narrative에 출력하지 않음 (최종 입원일 YYYY.M.DD 형식 예외)

13. **Affect(정동) 서술이 1~2문장으로 짧은 경우**:
    - 동일 시간 블록 내 다른 정동 표현(감정 변화, 강도, 신체 반응 등)을 추가 서술
    - 면담 기록에 정동 관련 표현이 부족한 경우: "[정동 추가 확인 필요]" 삽입 후 가용 정보로 최대 서술
    - meta.missing_items에 "A(Affect) 서술 보강 필요" 기재

14. **Gold Standard 오염 표현 감지 시**:
    - narrative 생성 후 §2 규칙 5의 금지 표현 목록과 자가 대조
    - 금지 표현이 포함된 경우: 현재 환자의 면담 데이터 기반 표현으로 교체
    - 면담 데이터에 대응 표현 없으면 해당 부분 "[확인 필요]"로 처리
    - meta.missing_items에 "GS 오염 표현 감지 및 수정" 기재
    - meta.requires_review = true 설정

### 에스컬레이션 규칙

  meta에 `"alert": "HIGH_SUICIDE_RISK"` 추가
